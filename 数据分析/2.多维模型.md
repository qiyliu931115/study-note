# Oracle 的王牌：Essbase

Oracle 收购 Hyperion 后得到的瑰宝。Essbase 几乎是多维数据库（MOLAP）的代名词。

特点：它把数据预先算好存成一个“大方块”。比如“2025年-华东区-销售部-第一季度”这个交叉点的数据，它早就已经在后台静默算好了。

优势：查询速度是毫秒级的。不管你怎么切片（Slice）、切块（Dice）、钻取（Drill down），它都快得离谱，非常适合财务搞预算和合并报表。

# IBM 的王牌：Cognos TM1 (现 Planning Analytics)

这是 IBM BI 体系的核心。它的多维模型是基于内存的。

优势：计算能力极强，特别是在这就改数、马上看结果（Write-back）的场景下，比 Oracle 的静态查询更灵活。

# Microsoft SSAS (SQL Server Analysis Services)：

很多企业的 BI 报表其实背后都是连的 SSAS 的 Cube，用 MDX (多维表达式) 语言来查询，而不是 SQL。
