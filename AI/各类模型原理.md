# 各类模型原理：Transformer、VLM、文生图、图生图、多模态

这份内容按 8 个章节组织。每个章节控制在一页 PPT 能讲清楚的长度：先给一句话结论，再用流程和要点说明核心原理。

---

## 1. 总览：几类模型分别解决什么问题

Transformer 是处理信息关系的基础结构；大语言模型（LLM）用它处理文字，视觉语言模型（VLM）用它连接图像和文字；文生图和图生图通常以扩散模型等生成模型为主体，Transformer 可以作为文本编码器或部分骨干参与其中。

可以先把几类模型放在同一张图里看。下面是按应用方向排列，不代表它们是一级一级派生出来的关系：

```text
Transformer
  ↓
大语言模型 LLM：理解文字、生成文字
  ↓
VLM：在文字之外，增加图片理解能力
  ↓
文生图：根据文字生成图片
  ↓
图生图：根据原图和文字生成新图片
```

它们的共同点是都要把输入转换成模型能计算的向量。文字会变成 token 向量，图片会变成视觉向量，模型再在这些向量之间建立关系。

不同点在于输出目标不同：LLM 主要输出文字，VLM 主要输出对图片的文字解释，文生图和图生图主要输出图片。

---

## 2. Transformer：让模型理解上下文关系

Transformer 的核心是 attention（注意力），让每个 token 自己决定该关注上下文里的哪些信息。

文本不会整句处理，而是先切成 token（词元）。下面按词切分只是演示，真实的 token 可能是字、词或更细的子词片段：

```text
我想回县城创业
  ↓
我 / 想 / 回 / 县城 / 创业
```

每个 token 再变成一串数字，也就是 embedding（嵌入向量）：

```text
县城 -> [0.12, -0.43, 0.88, ...]
创业 -> [0.36,  0.21, 0.17, ...]
```

光有 embedding 还不够，模型还要知道顺序，所以加入位置信息（positional encoding），才能区分“我喜欢你”和“你喜欢我”。

接下来 attention 会计算 token 之间的关系。例如：

```text
他把苹果放进包里，因为它太重了。
```

模型需要判断“它”指什么。Attention 做的就是让“它”去看上下文里的其他词，给每个词分配不同的关注程度。

```text
“它”关注：
苹果：较高
包：较高
太重：很高
他：较低
```

真实的 Transformer 内部用 Q/K/V、多头注意力（multi-head attention）和前馈网络（feed-forward network，FFN）完成这些计算。这些术语不必记，只要抓住一点：attention 是在动态计算“当前 token 该参考哪些上下文”。

整体流程可以概括为：

```text
输入文本
  ↓
切成 token
  ↓
变成 embedding
  ↓
加入位置信息
  ↓
attention 计算上下文关系
  ↓
多层反复加工，得到更抽象的上下文表示
  ↓
输出头把向量转成具体结果
```

最后的“输出头（output head）”像个转换器：Transformer 产出的是一组人读不懂的高级向量，输出头再把它转成具体结果。不同任务接不同输出头：生成任务预测下一个 token（叫语言模型头，language modeling head，简称 LM head），分类任务输出类别概率（classification head），embedding 任务输出一个可用于检索或相似度计算的向量。

Transformer 不是给每个词记住一个固定不变的含义，而是根据上下文动态计算词与词之间的关系；最终结果由这些上下文表示经过输出头转换得到。

## 3. LLM：大语言模型如何生成文字

上一章的输出头里，最常见的一种就是生成文字。大语言模型生成文字，本质上就是把这台 Transformer 反复运行，每次用输出头预测出下一个 token。

还用上一章那句「他把苹果放进包里，因为它太重了」——它也是模型一个词一个词写出来的。假设已经写到「他把苹果放进包里，因为它太」，要预测下一个词：

```text
他把苹果放进包里，因为它太 ___
```

用的就是 Transformer：把已有文字送进去，给词表里每个词（可能有几万个）打一个原始分数（logits），代表“它接在后面有多合适”，再用 softmax 变成总和为 100% 的概率：

```text
他把苹果放进包里，因为它太 ___
  ↓ 给词表里每个词打分
重     58%
沉     16%
大      9%
硬      3%
…（其余几万个词，概率都很低）
  ↓ 按概率选出一个
重
```

选出一个 token，接到原文后面，再把整段送回同一台 Transformer 预测下一个。如此反复，这个“边生成边接上”的过程叫自回归（autoregressive）：

```text
他把苹果放进包里，因为它太 重
他把苹果放进包里，因为它太重 了
他把苹果放进包里，因为它太重了 ，
他把苹果放进包里，因为它太重了， 于是
他把苹果放进包里，因为它太重了，于是 他
```

这就是模型看起来像在“写文章”“回答问题”的原因——它其实是在一步步生成符合上下文的后续内容。生成时不一定选概率最高的 token，温度和采样策略会影响答案的稳定性和多样性。

LLM 能回答问题，是因为它在大量文本中学到了语言模式、知识关联、表达方式和推理模板。但它不是数据库检索，也不是严格按规则执行程序。它的输出是概率生成结果，所以可能出现错误、编造或前后不一致。

## 4. VLM：让模型同时理解图片和文字

VLM 会把图片变成视觉向量，再和文字向量放到同一个模型里理解。

图片不能直接被模型“看见”，需要先转换成数字。常见做法是把图片切成很多小块（patch，图块），每个小块变成一个视觉向量：

```text
原始图片
  ↓
切成 patch
  ↓
每个 patch 变成视觉向量
  ↓
和文字 token 一起进入模型
```

例如用户上传一张农产品包装图，再问“生产日期是什么”。VLM 需要同时做几件事：

```text
识别图片里有哪些区域
识别区域中的文字
理解用户问题在问生产日期
找到对应位置
用文字回答
```

VLM 的关键是图文对齐（image-text alignment）。模型需要学会“图片里的这个区域”和“文字里的这个词”之间有什么关系，例如图片中的狗对应 dog，包装上的数字对应日期，表格里的列对应价格和数量。

因此，VLM 更像是给语言模型增加了视觉输入能力。它不只是 OCR 认字，还能理解物体、场景、空间关系和图文关系；它通常输出文字答案，而不是直接生成图片。

## 5. 文生图：文字如何变成图片

文生图（text-to-image）通常是把提示词（prompt）变成条件，再让生成模型从噪声中一步步生成图片。

用户输入一段提示词：

```text
一条乡村公路，两边是稻田，远处有青山，傍晚金色光线，写实摄影风格
```

模型会先用文本编码器（text encoder）把这段文字编码成文本向量：

```text
提示词
  ↓
文本编码器
  ↓
文本条件向量
```

很多文生图模型使用扩散模型（diffusion model）。它的生成过程可以理解为从一张随机噪声图开始，逐步去噪（denoising），最后得到符合提示词的图片：

```text
随机噪声
  ↓
根据提示词去噪
  ↓
出现大致构图
  ↓
形状、颜色、光照变清楚
  ↓
生成最终图片
```

提示词不是最后贴上去的标签，而是在每一步去噪时都参与引导。它会影响主体、场景、风格、构图、光线和细节。

文生图也不是简单从图库里搜索一张图。更准确地说，模型从训练数据中学到了图像规律和图文对应关系，然后生成一张新的图片。

## 6. 图生图：基于原图生成新图片

图生图（image-to-image）不是从纯噪声开始，而是把原图作为参考，再结合提示词生成新图片。

基本流程可以这样理解：

```text
输入原图
  ↓
加入一定程度的噪声
  ↓
结合提示词去噪
  ↓
输出新图片
```

如果噪声加得少，模型会更保留原图；如果噪声加得多，模型的发挥空间更大。

例如：

```text
原图：一张乡村院子的照片
提示词：改成冬天雪景，写实风格
输出：院子结构大体保留，但变成雪景
```

图生图常见用途包括风格转换、草图精修、背景替换、换季节、换材质、提升清晰度和局部重绘。

局部重绘会额外使用 mask（掩码），告诉模型哪里可以改、哪里要保留：

```text
原图 + mask + 提示词
  ↓
只重绘 mask 区域
```

如果需要更强的结构控制，还可以加入边缘图、深度图、姿态骨架等条件，让模型在生成时更严格地保留轮廓、姿势或空间关系。

## 7. 多模态模型：把多种输入放进同一个模型

多模态模型把文字、图片、声音、视频等不同类型的输入，统一变成向量放进同一个模型，既能理解也能生成。

前面几章其实各是一种局部的多模态能力：VLM 把图和文放在一起理解，文生图、图生图在文字和图片之间转换。多模态模型（multimodal model）的趋势，就是把这些能力合进一个模型。

做法和前面一脉相承：每种模态先用各自的编码器变成向量，一起进入同一个 Transformer，再按需要接不同输出头。

```text
文字 / 图片 / 声音 / 视频
  ↓ 各自编码成向量
进入同一个 Transformer
  ↓ 按需要接不同输出头
输出文字 / 图片 / 语音
```

例如：

```text
一段语音提问 + 一张图  ->  用语音回答
一段文字 + 一张草图    ->  生成成品图
一段视频             ->  用文字总结内容
```

实现上有两条路：早期多是“拼接式”——把语音转文字、LLM、文字转语音等几个专用模型串起来，各管一段；现在的趋势是“原生多模态（native multimodal）”——一个模型从一开始就用多种模态一起训练，端到端地既理解又生成，不再靠多个模型拼接。

不过多模态训练比纯文本明显更难：高质量的图文、音视频对齐数据远比文本稀缺，图片尤其视频展开成 token 后数量巨大、训练成本高，多种模态一起训还容易互相干扰、甚至把原有能力练退化。所以不少团队会先把资源押在文本上，多模态另立项目推进。

---

## 8. 容易混淆的点：理解模型和生成模型不同

VLM 主要负责理解图片并输出文字，文生图和图生图主要负责生成图片，它们不是同一种任务。

可以用输入输出区分：

```text
LLM：
文字 -> 文字

VLM：
图片 + 文字问题 -> 文字答案

文生图：
文字提示词 -> 图片

图生图：
原图 + 文字提示词 -> 新图片
```

Transformer 也不等于扩散模型。Transformer 擅长建模 token 或 patch 之间的关系，扩散模型擅长从噪声中逐步生成图像。现代模型经常把它们组合起来使用。

还要注意，提示词不是程序代码。模型会受提示词影响，但不会百分百严格执行。它可能漏掉细节、混淆对象、生成错误文字，或者在复杂结构上不稳定。

最终可以这样总结：

```text
Transformer：负责理解关系
LLM：用 Transformer 生成文字
VLM：把图片和文字放在一起理解
文生图：从文字条件生成图片
图生图：在原图基础上生成新图片
多模态模型：把多种模态放进一个模型，既理解又生成
```
